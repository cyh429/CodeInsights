# 2026-05-22 阶段 15 Agent Runner 链路手动切换

## 结论

阶段 15 已按用户确认完成 Agent Runner 链路手动切换：桌面 Agent 输入区底部工具栏新增紧凑按钮，后续发送可选择 `Runner v2` 或 `Legacy`。默认仍为 `Runner v2`，选择会持久化到 settings。

本阶段没有删除旧 Agent 主循环、Pipeline legacy adapter、旧 Feishu bridge 或旧 session JSONL 兼容。

版本：

- `@codeinsights/shared`: `0.1.41`
- `@codeinsights/electron`: `0.0.99`

## 实现范围

- `AgentSendInput.runtimeRunnerMode` 扩展为本次发送可选链路：`runner-v2` / `legacy`。
- `AppSettings.agentRuntimeRunnerMode` 持久化用户选择，启动时恢复到 Jotai atom。
- Agent 输入区底部工具栏在权限模式与思考按钮之间新增链路按钮，显示 `Runner v2` 或 `Legacy`。
- 主进程通过 `resolveAgentRuntimeRunnerModeForRun()` 解析实际链路：
  - 默认：`Runner v2`
  - 未显式设置 env 且 UI 选择 `Legacy`：旧 Agent 主循环
  - `CODEINSIGHTS_AGENT_RUNTIME_RUNNER_V2=0` / `false` / `off` / `no` / `disabled`：硬回滚旧 Agent 主循环
  - `CODEINSIGHTS_AGENT_RUNTIME_RUNNER_V2=1` / `true` / `on` / `yes` / `enabled`：强制 Runner v2
- `run_started.runnerMode` 记录本次实际链路，主进程日志输出 `Runtime Runner 链路: ...`。

- UI 切到 `Legacy` 可让后续桌面 Agent 发送走旧 Agent 主循环。
- 设置 `CODEINSIGHTS_AGENT_RUNTIME_RUNNER_V2=0` 可强制旧 Agent 主循环，并优先于 UI 选择。
- 旧 Agent 主循环保留，不删除旧 retry / Watchdog / Teams / SDKMessage 兼容路径。
- Pipeline legacy adapter、旧 Feishu bridge 和旧 session JSONL 兼容均未触碰。
