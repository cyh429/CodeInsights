## 本阶段完成项

### Runner v2 自动重试

- `InProcessAgentRuntimeRunner` 支持 retryable catch error 自动重试。
- 新增 retry 生命周期 runtime events：
  - `retry_scheduled`
  - `retry_attempt`
  - `retry_cleared`
  - `retry_failed`
- 重试成功后沿用同一 `runId`，并发出 `retry_cleared`。
- 重试耗尽后持久化 retry exhausted SDKMessage，并输出 `run_failed`。

### Typed Error / Catch Error 持久化

- assistant typed error 会映射并持久化 typed error SDKMessage。
- catch 分支错误会持久化用户可读错误 SDKMessage。
- prompt too long catch error 会映射为 `prompt_too_long`。

### UI sdk_message 推送

- Runner v2 deps 新增 `onSdkMessage`。
- Orchestrator Runner v2 path 通过 event bus 继续发送 `{ kind: "sdk_message", message }`。
- 聚焦测试已覆盖 assistant / result 均能推送到 UI。

### Event Log 去重

- Runner v2 path 会过滤 runner 自身输出的 `run_started` / `sdk_session`，避免 Orchestrator 已写入的 event log 出现重复前置事件。
- 2026-05-19 真实 approve 补跑发现 `queryOptions.onSessionId` 在 Runner v2 下仍会多次直接写入相同 `sdk_session`；已在 event log writer 层按同一 run 的 `sdkSessionId` 去重，并补充聚焦测试。

### Plan Mode 退出事件持久化

- 2026-05-19 真实 Plan Mode 补跑发现 Renderer 收到 `exit_plan_mode_resolved`，但 events JSONL 未持久化 `plan_mode_exited`。
- 已在 ExitPlanMode IPC 响应后写入 `plan_mode_exited`，并补充 event log 聚焦测试。
- 仅 `approve_auto` / `approve_edit` 写入 `plan_mode_exited`；`deny` / `feedback` 仍保持 plan mode active，不伪造退出事件。

### Watchdog / Teams auto-resume

- Runner v2 复用 `TeamsCoordinator` 追踪 `task_started` / `task_notification`，与旧主循环保持同一 Teams 状态来源。
- Runner v2 捕获 SDK session 后同步写入 coordinator，避免 auto-resume / watchdog 因缺少 `sdkSessionId` 静默失效。
- Teams 活跃时 Runner v2 会延迟 `result` 的 UI 推送，在 resume query 完成后再发送，避免前端提前把 teammate 标为 stopped。
- Runner v2 通过 legacy lifecycle 回调继续向 Orchestrator 发送 `waiting_resume` / `resume_start`，保持 UI 副作用路径不变。
- Watchdog tick 会调用 `areWorkersIdle()`；当 worker 已 idle 而 query 仍挂起时，Runner v2 退出当前 query loop 并进入后续收尾 / auto-resume 流程。
