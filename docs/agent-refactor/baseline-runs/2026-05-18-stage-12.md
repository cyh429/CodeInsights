### 发现并修复

- 问题：旧 Agent 主循环在 permission pending 状态下调用 `stopAgent()` 时，SDK iterator 可能正常结束而不是抛 abort 错误；旧逻辑会清掉 pending 和 active 状态，但不写 `run_stopped` 终态。
- 修复：在 iterator 正常结束后、进入“正常完成”前检查 `activeSessions`；如果已被 stop，持久化已累积 SDKMessage，写 `run_stopped`，发送 `stoppedByUser` completion。
- 追加加固：代码审查后发现 Runner v2 正常结束路径存在同类风险，已在 `runWithRuntimeRunnerV2()` 结束后复验 `activeSessions` 并补写 `run_stopped`。
- 加固：`stop(sessionId)` 只在 session 确实 active 时写入 `stoppedBySessions`，避免对已完成会话 stop 后留下 stale 中止标记。
- 回归测试：`apps/electron/src/main/lib/agent-orchestrator/completion-signal.test.ts` 新增 “用户中止后 iterator 正常结束时也会补写 run_stopped” 与 “Runner v2 用户中止后正常结束时也会补写 run_stopped”。
